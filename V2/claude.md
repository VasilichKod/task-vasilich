# Task Vasilich V2

## Что это за версия

`V2` это альтернативная модель недельного планировщика по сравнению с основной версией в корне `TaskVasilich`.

Главная идея `V2`:
- группы остаются вертикальными слева
- внутри каждого дня задачи сгруппированы по проектам
- проект отображается прямо в ячейке дня, а не отдельной строкой слева на всю неделю
- пользователь смотрит на конкретный день и сразу видит все проекты и задачи этого дня

Файлы версии:
- `version2.html`
- `style-v2.css`
- `app-v2.js`


## Основная логика отображения

### Структура таблицы

- столбцы это дни недели: `Пн ... Вс`
- слева отдельный узкий вертикальный столбец групп
- каждая группа занимает одну строку таблицы
- в каждой ячейке дня внутри группы отображается список проектных карточек

### Проект в дне

Каждая карточка проекта внутри дня содержит:
- название проекта
- крестик удаления проекта только из конкретного дня
- список задач проекта в этом дне
- тихую строку быстрого ввода задачи внутри карточки

Визуально карточка проекта сейчас устроена так:
- белый фон
- без рамки по периметру
- без скругления
- только верхняя линия
- верхняя линия окрашена в цвет проекта
- цветные кружки в недельной сетке убраны

Крестик удаления проекта:
- скрыт по умолчанию
- появляется плавно только при наведении на верхнюю полоску проекта
- при наведении на сами задачи ниже не появляется

Удаление проекта из дня:
- удаляет проект только из этого дня
- сам проект из системы не удаляется
- задачи проекта в других днях и неделях не затрагиваются


## Добавление задач

Сейчас есть 2 способа добавить задачу.

### 1. Верхняя кнопка `+ задача`

Открывает отдельное окно создания задачи.

Пользователь выбирает:
- группу
- день
- проект
- текст задачи

Если проект ещё не отображается в выбранном дне:
- он автоматически добавляется в этот день
- затем туда создаётся задача

### 2. Тихий инлайн-ввод внутри карточки проекта

Внизу карточки проекта есть почти невидимая кликабельная строка без текста и иконок.

Поведение:
- по нажатию открывается компактный инлайн-ввод
- `Enter` создаёт задачу
- если пользователь вводит текст и уводит фокус, задача тоже создаётся
- `Esc` закрывает ввод без сохранения

Это сделано для быстрого ввода на ПК без визуального шума.


## Работа с задачами

### Что можно делать с задачей

- отметить выполненной чекбоксом
- открыть попап задачи нажатием на текст
- изменить название
- изменить заметку
- перенести на другой день
- удалить через попап

### Что убрано специально

В недельном отображении убраны:
- цветные кружки у проектов
- крестики удаления у каждой отдельной задачи

Причина:
- уменьшить визуальный шум
- оставить удаление задачи только через попап

### Попап задачи

В попапе задачи есть:
- поле названия
- заметка
- выбор дня
- `Удалить`
- `Выполнено / Не выполнено`
- `В график` для задач со страницы `Задачи`
- `Сохранить`

Попап закрывается:
- по крестику
- по клику вне окна


## Drag and drop

Обычные задачи можно перетаскивать:
- между проектами
- между днями

Если задача переносится в проект, который ещё не показан в выбранном дне:
- проект автоматически появляется в этом дне

Постоянные задачи не перетаскиваются.

### Drag and drop проектов

Проекты тоже можно перетаскивать мышкой.

Сейчас поддержано:
- перенос проекта в другой день внутри текущей недели
- перенос проекта вместе с обычными задачами этого дня
- перестановка проектов местами внутри одного дня

Что важно:
- перенос работает только внутри своей группы
- если проект переносится в другой день, вместе с ним переносятся обычные задачи этого дня
- постоянные задачи не переносятся вместе с блоком, потому что они живут по отдельной шаблонной логике


## Добавление проектов в день

Кнопки `+ проект` внутри каждой ячейки убраны специально, чтобы таблица выглядела чище.

Теперь проект в конкретный день добавляется через верхнюю кнопку `+ проект`.

В окне выбора задаются:
- группа
- день
- проект

Список проектов зависит от выбранной группы и дня:
- показываются только проекты этой группы
- уже добавленные в этот день проекты не предлагаются повторно

Проекты можно также переставлять drag-and-drop уже после добавления в день.


## Шаблон проектов

Есть отдельное окно `Шаблон проектов`.

Назначение:
- задаёт, какие проекты должны автоматически появляться в новой неделе по дням
- это замена старой идеи “все проекты на каждую неделю”

Важно:
- шаблон влияет на новые недели
- уже вручную изменённые недели не пересобираются автоматически

### В шаблоне можно

По каждому проекту:
- `Всё` — включить проект на все дни
- `Снять` — убрать проект со всех дней

По каждой группе:
- `Добавить всё` — включить все проекты группы на все дни
- `Убрать всё` — снять все проекты группы со всех дней

Также можно вручную ставить галочки по отдельным дням.


## Постоянные задачи

Есть отдельное окно `Постоянные задачи`.

Оно позволяет:
- создавать постоянные задачи
- редактировать их
- удалять их
- фильтровать по группе
- фильтровать по проекту

Логика постоянных задач:
- они автоматически отображаются в нужном проекте и дне каждой недели
- статус выполнения хранится отдельно для каждой недели
- заметка тоже хранится на уровне конкретной недели

То есть сама постоянная задача шаблонная, а выполнение недельное.


## Sidebar

### Полный sidebar

В полной версии sidebar содержит:
- навигацию
- блок справочников
- системный блок
- сводку

Текущая структура меню:

Навигация:
- `График`
- `Задачи`
- `Достижения`
- `История и аналитика`

Справочники:
- `Проекты`
- `Группы`
- `Шаблоны`
- `Постоянные задачи`

`Проекты` и `Группы` раскрываются прямо внутри блока `Справочники`, а не отдельными дублирующими секциями ниже.

Система:
- `Настройки`

### Сворачивание sidebar

Sidebar можно свернуть в узкую полоску.

В свернутом режиме:
- остаются только иконки основных пунктов
- текст скрывается
- раскрывающиеся справочники скрываются полностью

Кнопка сворачивания:
- `⇤` — свернуть
- `⇥` — развернуть

### Мобильная версия

На мобильной ширине sidebar работает как выезжающее burger-меню.

### Системные страницы

В блоке `Система` сейчас есть:
- `Профиль`
- `Настройки`

`Профиль`:
- локальные данные пользователя
- имя
- email
- роль
- город
- описание
- статус аккаунта как задел под будущую авторизацию и синхронизацию

`Настройки`:
- название рабочего пространства
- стартовая страница сервиса
- сворачивать ли sidebar при старте
- открывать ли достижения сразу на текущем году
- блок резервной копии данных
- сервисный блок под будущую серверную версию


## Группы и проекты

### Группы

Через sidebar можно:
- добавлять группу
- редактировать название группы
- менять цвет группы
- удалять группу

Ограничение:
- нельзя удалить группу, если в ней ещё есть проекты
- должна остаться хотя бы одна группа

### Проекты

Через sidebar можно:
- добавлять проект
- редактировать название
- менять группу
- менять цвет
- удалять проект

При удалении проекта:
- удаляются его обычные задачи
- удаляются его задачи со страницы `Задачи`
- удаляются его достижения по всем годам
- удаляются его постоянные задачи
- удаляются его weekly-статусы для постоянных задач
- проект убирается из day view и из шаблона проектов


## Страница Задачи

`Задачи` это отдельный слой над недельным графиком.

Логика страницы:
- здесь лежат проектные задачи без привязки к конкретному дню
- это общий список идей, целей и задач по проекту
- потом отдельные задачи можно отправлять в текущую неделю через кнопку `В график`

Что есть на странице:
- группы
- карточки проектов по 3 в ряд
- у каждой карточки только верхняя цветная линия
- фон чуть серее, как заметка
- проект можно скрыть крестиком
- скрытый проект можно вернуть через верхнюю кнопку `+ проект`
- внизу проекта тихий инлайн-ввод задачи

Добавление задач:
- сверху через `+ задача`
- или тихой строкой прямо внутри карточки проекта

Из попапа задачи можно:
- редактировать текст
- редактировать заметку
- удалить задачу
- отметить выполненной
- отправить задачу в неделю через `В график`


## Страница Достижения

`Достижения` это архив сильных результатов по проектам.

Основная логика:
- достижения хранятся по годам
- внутри года разбиваются по группам и проектам
- у каждой записи есть дата и текст достижения

Что есть на странице:
- фильтр `Все` или конкретный год
- в режиме `Все` годы можно сворачивать и разворачивать
- верхняя кнопка `+ достижение`
- верхняя кнопка `+ год`
- верхняя кнопка `+ проект`

Проекты на странице достижений:
- можно скрывать проект внутри конкретного года
- можно возвращать его обратно через `+ проект`
- скрытие проекта не удаляет сами достижения, а только убирает отображение проекта в этом году

Запись достижения:
- год
- группа
- проект
- дата
- текст достижения

Редактирование:
- запись открывается кликом по достижению
- можно редактировать
- можно удалять


## Ширина колонок

В `V2` можно менять ширину каждого дневного столбца отдельно.

Ограничения:
- минимум `160px`
- максимум `360px`
- значение сохраняется в `localStorage`

Это сделано отдельно для каждого дня, а не одной общей шириной на всю неделю.


## Стили и визуальный язык

`V2` приведена к стилю основной версии:
- молочный светлый фон
- спокойные серые поверхности
- графитовый sidebar
- мягкие границы
- минимум лишних цветных маркеров

Осознанно убрано или приглушено:
- лишние иконки
- цветные точки в недельной сетке
- крестики у задач
- кнопки `+ проект` в каждой ячейке

Текущее состояние карточек проектов:
- ровные края без скругления
- белый фон
- нет рамки по периметру
- есть только верхняя линия
- верхняя линия окрашена в цвет проекта
- крестик проекта появляется только на хедере при наведении
- цветные кружки в недельной сетке убраны

Идея `V2`:
- меньше шума
- больше ощущения чистого листа
- внимание на содержимом дня, а не на интерфейсном мусоре

## Важные отличия V2 от V1

- в `V1` проекты идут по строкам
- в `V2` проекты живут внутри конкретных дней

- в `V1` удобно смотреть проект по всей неделе
- в `V2` удобно смотреть один день целиком

- `V2` специально чистится от лишних кнопок и визуального шума
- `V1` пока остаётся более “панельной” и плотной по управлению


## Что уже специально решено по UX

- удаление задачи только через попап
- проект добавляется сверху, а не кнопками в каждой ячейке
- задача может добавляться и сверху, и тихо прямо внутри карточки
- шаблон проектов вынесен отдельно, чтобы новая неделя не выглядела кашей
- sidebar можно сворачивать в узкую полосу
- проекты можно переносить мышкой по дням и менять их порядок без отдельных кнопок
- верхняя панель `Графика` сейчас содержит:
- `+ задача`
- `+ проект`
- `Перенос задач`
- `Перенос задач` вынесен из sidebar в верхнюю панель страницы `График`
- `ИИ-анализ` не выделяется отдельной кнопкой в верхней панели и дальше должен жить внутри раздела `История и аналитика`
- страница `Задачи` это отдельный backlog по проектам без дней
- страница `Достижения` это архив по годам и проектам
- `Профиль` и `Настройки` уже существуют как каркас будущего сервиса


## Экспорт и импорт

В `Настройках` есть блок `Данные`.

Сейчас поддержано:
- `Экспорт JSON`
- `Импорт JSON`

Экспорт:
- сохраняет всю локальную базу сервиса в один `.json`

Импорт:
- загружает `.json`
- нормализует данные
- подменяет текущее локальное состояние
- сразу перерисовывает интерфейс

Это уже задел под:
- резервные копии
- перенос между браузерами
- будущую серверную синхронизацию
