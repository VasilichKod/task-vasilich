# WeekPlanner — суть проекта

## Идея

Я давно пользуюсь Google Таблицей как таск-менеджером — вся неделя на одном экране,
строки по категориям (Работа, Семья, Дом...), столбцы по дням. Просто, быстро, наглядно.

Все остальные инструменты (Notion, Todoist, Trello) неудобны тем,
что нельзя охватить взглядом сразу всё — приходится кликать, переключаться, листать.

**Цель** — сохранить этот принцип "вся неделя на одном экране", но сделать
полноценный веб-сервис: с историей недель, ИИ-анализом и синхронизацией между устройствами.

---

## Ключевые принципы

- Вся неделя — один экран, без лишних кликов
- Каждый проект / категория — отдельная строка (не костыль внутри ячейки)
- Фильтр: Всё / Работа / Личное / конкретный проект
- ИИ смотрит на неделю и даёт короткий анализ — что сделано, что зависло, совет
- История не стирается — можно листать прошлые недели и видеть паттерны

---

## Структура данных

```
Группа (Работа / Личное)
  └── Проект / Категория (4Станка, SHMEL, Семья, Спорт...)
        └── Задача [день недели] [выполнено / нет] [заметка]
```

Проекты добавляются вручную, каждому задаётся цвет и группа.

## Текущий UX

- `+` в ячейке не открывает попап, а даёт инлайн-поле ввода прямо в таблице
- Клик по готовой задаче открывает окно задачи
- В окне задачи можно менять текст, день и заметку
- Попапы закрываются кликом вне окна
- Проекты редактируются из sidebar

**Файлы:** `index.html` + `app.js` + `style.css` + `сlaude.md` — всё открывается без сервера.

## Как запустить сейчас

1. Скачать файлы проекта, положить в одну папку
2. Открыть `index.html` в браузере — всё работает
3. Для ИИ-анализа: вставить ключ Anthropic в первую строку `app.js`
   ```js
   const ANTHROPIC_API_KEY = 'sk-ant-...';
   ```
   Ключ получить на [console.anthropic.com](https://console.anthropic.com)

## Что хотим в итоге

### Этап 2 — Бэкенд + база данных
- Node.js + Express + PostgreSQL
- Данные хранятся на сервере, а не в браузере
- ИИ-ключ на сервере (безопасно)
- API эндпоинты: задачи, проекты, анализ
- Хостинг: Railway / Render (~$5/мес)
- Добавить нормальную модель для недель, проектов, задач и заметок
- Подготовить серверный перенос задач между неделями без дублей

### Этап 3 — Авторизация
- Magic link по email (без паролей) или Supabase Auth
- Один аккаунт — компьютер и телефон синхронизированы

### Этап 4 — PWA
- Два файла (`manifest.json` + `sw.js`) → приложение устанавливается на телефон
- Работает как нативное, без App Store

### Этап 5 — Умный ИИ (когда накопится история)
- Сравнение с прошлыми неделями
- "SHMEL застревает каждый вторник уже 3 недели — перенести?"
- Предложения по перераспределению нагрузки
- Еженедельный дайджест (например, в Telegram)
